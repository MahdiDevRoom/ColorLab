ColorLab.js
Documents

ColorLab.js is a lightweight script that provides color tools for designers, such as color mixing and color harmonies.

Installation

To install, simply add the class code to your project, or download the script and include it in your HTML document. 


// Include the script in your HTML:
<script src="path/to/ColorLab.js"></script>

Now call it:


// Example usage:
const C = new ColorLab();
console.log(C.version); // '1.0.0'

Helper Methods

normalize()
Helper Methods

Normalizes any supported color into a consistent output (or undefined for invalid inputs).


C.normalize('medium SPRiNG green'); // 'Medium Spring Green'
C.normalize('hsl(129, 80, 56)'); // 'hsl(129deg 80% 56% / 1)'
C.normalize('#f02'); // '#FF0022FF'

getChannels()
Helper Methods

Returns channel values for the detected format (keyword returns undefined).


C.getChannels('#244A8D'); // { R: '24', G: '4A', B: '8D', A: 'FF' }
C.getChannels('hsl(218 59% 35%)'); // { H: 218, S: 59, L: 35, A: 1 }

isColor()
Helper Methods

Returns true for valid color inputs, otherwise false.


C.isColor('#ff00aa'); // true
C.isColor('apple'); // false

detect()
Helper Methods

Detects input color type: "keyword", "hex", "rgb", "hsl", "hsv", "cmyk", or undefined. 


C.detect('royal blue'); // 'keyword'
C.detect('#f02'); // 'hex'
C.detect('rgb(10 20 30)'); // 'rgb'

randomColor()

Generates a random HEX color.

Utilities & Properties
convert, channels


convert
Utilities & Properties

The convert getter is a powerful, smart API for color transformations. It automatically detects the input color format and allows converting it to any other supported format. The supported formats are: keyword, hex, rgb, hsl, hsv, and cmyk.

You can use convert.to.<format> for automatic detection and conversion, or directly call convert.<from>.<to> to specify source and target formats. When the source and target formats are the same (e.g., convert.hsl.hsl()), the method normalizes the color to a standard valid string. 


// Automatic detection & conversion:
C.convert.to.hex('rgb(255 128 0)');      // '#FF8000'
C.convert.to.rgb('#FF8000');             // 'rgb(255 128 0)'

// Explicit from-to conversion:
C.convert.rgb.hex('rgb(255 128 0)');     // '#FF8000'
C.convert.hex.rgb('#FF8000');            // 'rgb(255 128 0)'

// Normalizing a color:
C.convert.hsl.hsl('hsl(120, 50, 50)');   // 'hsl(120 50% 50%)'
C.convert.rgb.rgb('rgba(255 24 200 / 0.4)');   // 'rgb(255 24 200 / 0.4)'

Notes:

  • convert.to.<format> automatically detects the input type and applies the correct conversion.
  • Calling convert.<from>.<from> (same format) returns a normalized color string.
  • Alpha values are preserved where applicable.
  • Invalid colors return undefined.

channels
Utilities & Properties

The channels getter provides helpers for constructing colors from individual channels. You can create colors in hex, rgb, hsl, hsv, and cmyk. All outputs are automatically normalized. 


// Build color from RGB channels:
C.channels.rgb.hex(255, 128, 0);          // '#FF8000'
C.channels.rgb.rgb(255, 128, 0);          // 'rgb(255 128 0)'

// Build color from HSL channels:
C.channels.hsl.hex(120, 50, 50);          // '#40BF40'
C.channels.hsl.hsl(120, 50, 50);          // 'hsl(120 50% 50%)'

// Build color from CMYK channels:
C.channels.cmyk.rgb(0, 50, 100, 0);       // 'rgb(255 128 0)'
C.channels.cmyk.hex(0, 50, 100, 0);       // '#FF8000'

Notes:

  • Channels are automatically clamped to valid ranges.
  • Alpha channel is optional and defaults to fully opaque.
  • Output strings are compatible with convert for further transformations.
